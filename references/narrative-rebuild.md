# Narrative Rebuild

## Use this file when

- the paper repeats the same binary contrast across title, contribution list, sections, experiments, and keywords;
- section names already read like mini-conclusions;
- the current draft explains the work in the order it was built rather than the order a reviewer should understand it.

## Procedure

1. Write a single identity sentence.
   State the paper's object, intervention, and evaluation frame in one sentence.
2. Name the central decision.
   Reduce the paper to one reviewable question such as "what changes the action boundary?" or "when is a ranking stable under a shared protocol family?"
3. Sequence the evidence.
   Use a compact path such as:
   stakes -> structural ambiguity -> exact object or audit protocol -> computation or mechanism -> empirical pattern -> boundary of inference.
4. Refit the headings to that path.
   Prefer descriptive section names over verdict-like names.
5. Break forced symmetry.
   If one contribution is the conceptual center and the others are supporting machinery or validation, let the prose reflect that.

## Common rewrite moves

- Collapse mirrored contribution bullets into one lead contribution paragraph plus shorter support claims.
- Replace repeated slogan words with domain-specific nouns that point to the actual object of study.
- Keep the same core question across abstract, introduction, experiments, and conclusion, but do not force each surface to repeat the same phrase.
- Let experimental questions vary in granularity; they do not need a one-to-one mapping with top-level contributions.

## Warning signs

- Every heading encodes the same contrast with slightly different nouns.
- The contribution list has perfectly parallel syntax but uneven evidentiary weight.
- The supplement carries definitions or results that the abstract and conclusion rely on.
