# Metadata Surfaces

## Scope

Use this file for the title, abstract, TL;DR, keywords, and OpenReview registration text.

## Abstract rules

- Lead with the problem object and the paper's exact intervention.
- Keep the number of headline numeric facts low. Prefer three or four numeric anchors, not a dense sequence of measurements.
- If two numbers tell the same story, keep the pair that shows the before/after shift and remove the rest.
- Convert less important counts into prose when the abstract is already numerically saturated.
- Keep robustness statements scoped to the tested family, control, or stress condition.

## TL;DR rules

- Treat the TL;DR as one sentence that states the audited object, the main empirical fact, and the setting.
- Stay below the venue's hard character limit when one exists.
- Omit setup details, caveats, and secondary diagnostics unless they are the actual headline.

## Keywords

- Favor retrieval-oriented technical terms over internal slogan words.
- Include the underlying task, metric, method family, and inferential frame when possible.

## OpenReview text

- Keep Title, TL;DR, and Abstract semantically aligned, but do not copy one string into all three slots.
- Use subject areas that match the paper's technical center rather than every possible downstream application.
- Keep manual-entry reminders separate from the text intended for upload.
